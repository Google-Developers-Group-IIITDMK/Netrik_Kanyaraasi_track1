📊 Data Flow

1

User Input

User clicks "Run AI Analysis" → Streamlit UI captures configuration (LLM enabled, fast mode)

2

Data Loading

Data Loader reads CSV files → Creates 1200 Candidate objects → Builds Job Description

3

Agent Initialization

AI-Native Agent initialized → Leave balances created for all employees → LLM Manager configured

4

Resume Screening

Ranking Engine scores candidates → Optional LLM semantic boost → Sorted by match score

5

AI Insights Generation

Executive Summary created → Cross-candidate patterns identified → Predictive recommendations generated

6

Results Display

UI renders insights → Top candidates shown → All 6 modules accessible via tabs

7

Export

User downloads JSON → Standardized format → All data included (pipeline, schedule, leave, escalations)

🎯 Six Core Modules

1. Resume Screening

Purpose: Rank candidates by match score

  • Deterministic scoring (60-10-20-10)
  • Explainable ranking
  • Reasoning chains
  • Fast mode: 2-5s for 1200 candidates

2. Interview Scheduling

Purpose: Assign interview slots

  • Constraint-safe allocation
  • Zero conflicts guaranteed
  • Slot validation
  • Pipeline state update

3. Questionnaire Generation

Purpose: Create personalized questions

  • Technical questions
  • Behavioral (STAR format)
  • Situational scenarios
  • Role-specific focus

4. Pipeline Management

Purpose: Track candidate status

  • State machine validation
  • Transition logging
  • Terminal state handling
  • Complete audit trail

5. Leave Management

Purpose: Process leave requests

  • Policy enforcement
  • Balance tracking
  • No underflow
  • Clear denial reasons

6. Escalation Handling

Purpose: Categorize query priority

  • Keyword-based classification
  • High/Medium/Low priority
  • Reasoning provided
  • Recommended actions
